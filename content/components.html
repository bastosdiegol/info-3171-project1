Components

Menus

Menus help organize navigation within a dashboard, guiding users to specific sections. Style active menu items to show the current page, and include hover effects for better user experience. Example usage: navigating between "Dashboard", "Reports", and "Settings" sections.

Dashboard Reports Settings Disabled 
    
<nav class="nav nav-pills mb-4">
  <a class="nav-link active" href="#">Dashboard</a>
  <a class="nav-link" href="#">Reports</a>
  <a class="nav-link" href="#">Settings</a>
  <a class="nav-link disabled" href="#">Disabled</a>
</nav>

Buttons

Buttons are interactive elements that trigger actions. In dashboards, they are often used for submitting data, navigating, or initiating tasks like "Download Report" or "Generate Chart". Customize each button style for primary actions, secondary actions, and success indicators. Add hover effects and focus styles to enhance accessibility. 

    
<div class="mb-3">
  <button class="btn btn-primary">Primary Action</button>
  <button class="btn btn-secondary">Secondary Action</button>
  <button class="btn btn-success">Save Changes</button>
</div>

Checkboxes

Checkboxes allow users to select multiple options at once, useful in dashboards for filtering data or customizing visible metrics. Below are examples of how checkboxes can be used to filter data types and select metrics in a dashboard interface.

Example: In a financial dashboard, checkboxes let users control which data types they want to view, such as "Sales," "Revenue," or "Expenses." Selecting multiple options allows a customizable data view where users see only the metrics they need. 

    
<div class="mb-3">
  <label class="form-check-label me-2">Show Data Types:</label>
  <label>
    <input type="checkbox" class="form-check-input"> Sales
  </label>
  <label>
    <input type="checkbox" class="form-check-input"> Revenue
  </label>
  <label>
    <input type="checkbox" class="form-check-input"> Expenses
  </label>
</div>

Radio Buttons

Radio buttons are used when only one option is allowed at a time. In dashboards, they can be used to toggle between different views or reporting options. For example, users might select between daily, weekly, and monthly views of a metric.

Example: In a data analytics dashboard, radio buttons allow the user to select a single time range for data visualization, such as "Daily," "Weekly," or "Monthly" views. This ensures that only one range is active, providing clarity and preventing data overlap in the displayed charts. 

    
<div class="mb-3">
  <label class="form-check-label me-2">Time Range:</label>
  <label>
    <input type="radio" class="form-check-input" name="timeRange" value="daily"> Daily
  </label>
  <label>
    <input type="radio" class="form-check-input" name="timeRange" value="weekly"> Weekly
  </label>
  <label>
    <input type="radio" class="form-check-input" name="timeRange" value="monthly"> Monthly
  </label>
</div>

Textboxes

Textboxes are used for user input, such as filtering data or searching for specific metrics. Ensure the placeholder text is informative to guide users. Customize textboxes for better visibility in data-driven dashboards. Example usage: entering keywords to filter through extensive datasets.

Dropdowns

Dropdowns allow users to select from multiple options without cluttering the interface. In a dashboard, they can be used to filter data views, such as selecting time periods or data categories. Customize the dropdown to be clearly identifiable and ensure options are relevant to the context.

  • Last 7 Days
  • Last 30 Days
  • Year to Date
    
<div class="dropdown mb-4">
  <button class="btn btn-secondary dropdown-toggle" type="button" data-bs-toggle="dropdown" aria-expanded="false">
    Select Data Range
  </button>
  <ul class="dropdown-menu">
    <li><a class="dropdown-item" href="#">Last 7 Days</a></li>
    <li><a class="dropdown-item" href="#">Last 30 Days</a></li>
    <li><a class="dropdown-item" href="#">Year to Date</a></li>
  </ul>
</div>

Tooltips

Tooltips provide additional information without taking up space. In dashboards, they are often used on data points in charts to show precise values or details on hover. Customize tooltips to be contextually relevant, with clear formatting and appropriate positioning. 

    
<div class="mb-4">
  <button
          type="button"
          class="btn btn-info"
          data-bs-toggle="tooltip"
          data-bs-custom-class="custom-tooltip"
          data-bs-placement="top"
          title="More info about this data point">
    Data Point Info
  </button>
</div>

Accordions

Accordions allow users to expand and collapse sections, perfect for organizing dense information like settings, report options, or data insights. Customize to support both expand and collapse states and ensure smooth animations for better usability. Each section should be clearly labeled to help users find specific information quickly.

Customize report filters, such as date ranges, metrics, and categories. Perfect for creating tailored insights.

Control display settings, such as light/dark modes, chart types, and layout configurations.

Configure advanced settings, such as API connections, data refresh intervals, and custom calculations.
